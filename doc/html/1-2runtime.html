6.1 Mu Runtime Symbols

for version 0.0.4

Types

The libmu library implements a 64 bit staged low tag pointer and can accommodate an address space up to 62 bits.

Immediate types fit their values in the tagged pointer, immediate strings/keywords are limited to 7 characters. Other types are allocated from the heap, and their heap offset is in the upper bits of the tagged pointer.

See src/libmu/core/type.h

  fixnum         62 bit signed integer
  float          32 bit single float IEEE 754
  keyword         7 byte keyword symbol
  string          7 byte string

Type symbols as returned from type-of are keywords:

Namespaces and Symbols

Namespaces map the symbol reference reader syntax to a symbol.

  ns:name         an external symbol in the ns namespace
  ns::name        an internal symbol in the ns namespace
  name            a symbol in the null (lexical) namespace
  :name           a keyword symbol

Symbols

All symbols are immutable and therefore constant. Symbols contain only a value.

Constant Symbols

See src/libmu/core/type.h See src/libmu/core/core-env.cc *

  :t             boolean true
  :nil           boolean false, empty list

  std-in         stream
  std-out        stream
  std-err        stream

Keyword Symbols

Keyword symbols evaluate to themselves.

Function/Special Forms

Function/special call forms are compiled by the intrinsic compiler. It allows the :lambda special form in the function position, which must otherwise be a symbol or an extant function object. All arguments to function/special calls are compiled.

Special Forms

See src/libmu/core/compile.cc

libmu special forms are named by keywords.

  :const         bind constant symbol
  :if            conditional
  :lambda        define fixed-arity function
  :quote.        quoted form, defer evaluation

Functions in the mu Namespace

libmu function arguments are fixed in arity and type. Functions that normally have rest arguments take a list of excess arguments as a separate argument. mu:funcall, for example, takes a function and a list as arguments.
