1. The mu Environment

mu is the native execution environment for the eko system and consists of the mu-runtime binary and runtime libraries libmu.so and libmu.a. mu-runtime is a small driver program that implements a source file loader and minimal break loop on top of the libmu runtime.

Libmu

The mu runtime is written in C++20, requires only the standard library, and supports traditional Lisp-style data types in a green-threaded, garbage collected environment. libmu implements 64 bit tagged pointers and can accommodate an address space up to 62 bits.

libmu extends a C++ API for embedded applications and is an evaluator for the mu language. For details on the API, see the 6.1 Libmu Runtime appendix.

libmu was designed to be minimalist to an extreme and provides only the facilities that support the evolution of higher-level languages. As a result, the libmu runtime supplies only a few special forms and functions.

libmu provides a reader for converting character strings/streams into mu expressions, an evaluator for operating on those expressions, and a printer for converting mu expressions to a character string/stream.

Mu Language

The mu language is an immutable namespaced Lisp-1 and provides fixed-width numeric types, lists, fixed-arity lambdas, exceptions, streams, threads, and specialized vectors. For details on the symbols and data types supported by libmu, see the Libmu Runtime and Libmu Symbol Reference appendices.

The mu language is deliberately minimal, including only the operations necessary to allocate data types from and maintain the heap, define and apply functions and access symbol values, access the lexical and dynamic environments, enable stream I/O, and implement a primitive read-eval-print loop.

In general, libmu contains functions that isolate the programmer from the details of the heap layout, though some concessions have been made for high-frequency and performance sensitive operations like list and vector element access. Special forms are limited to lambda, if, quote and a defining form for constant symbols.

There is a quick reference card for the mu namespace, see doc/refcards/mu-ref.[pdf,docx]

Mu Symbols and Keywords

mu symbols reside in a namespace. A namespace is a collection of related symbols, which are divided into external and internal symbols (designated by the “:” and “::” syntax, see below). An external symbol is meant to be easily accessible by other namespaces, and internal symbols private to the containing namespace (though the distinction can be somewhat arbitrary).

A keyword is a special symbol that evaluates to itself. It resides in the nil namespace along with unqualified symbols.

The mu symbols for the true and false boolean values are keywords, :t and :nil respectively. The mu reader interprets the empty list syntax () as :nil, is of type :symbol, and is true for the mu:keyword predicate.

The mu reader accepts unqualified symbols, these are almost always lexical symbols. Unqualified symbols reside in the nil namespace. Qualified symbols are used to name symbols that reside in a non-nil namespace and as such must be fully qualified to the mu reader.

Mu Expression Syntax

An abbreviated (and incomplete) BNF:

expr = atom | cons | list
atom = number | symbol | vector
cons = "(" expr "." expr ")"
list = "(" expr "." list ")" | "()"
symbol = namespace name | keyword
keyword = ":" name
namespace = name ":" | name "::" | ""
name = constitutent name
vector = "#(" vector-type exprs ")"
vector-type = :t | :byte | :fixnum | :float | :double
exprs = expr exprs | ""
